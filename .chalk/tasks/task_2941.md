## Context

Gemini review on PR #10 flagged (security-high) that `workspace.ts` embedded the
installation token in the git remote URL passed to `git remote add`, causing:
1. Process-list exposure (`ps aux` / `/proc/<pid>/cmdline`).
2. On-disk persistence in `.git/config` between fetch and `stripCredentials` —
   a crash/kill in that window strands a live token on the host.

The finding is legitimate and aligns with Magpie's structural-defense principle,
so I adopted it.

## Plan

- [x] `origin` now added with a **tokenless** URL (`https://github.com/o/r.git`).
- [x] Token supplied out-of-band via `GIT_TOKEN` env + ephemeral
      `-c credential.helper` snippet (echoes `$GIT_TOKEN`); never a CLI arg,
      never written to `.git/config`.
- [x] Credential helper applied to **both** fetch and checkout (blobless clone
      fetches missing blobs on demand during checkout — that also needs auth).
- [x] Simplified `stripCredentials` to FETCH_HEAD/logs deletion (defense in
      depth; origin is already tokenless). Removed now-dead `stripUrlCredentials`.
- [x] Updated all doc comments to the stronger "token never touches disk" model.

## Review

**Caught a real bug while verifying with live git:** an unqualified
`-c credential.helper=...` does NOT win. Git consults helpers in config order
and stops at the first that returns a password, so the host's ambient global
helper (e.g. `gh`'s / a keychain helper) answered first with the *wrong*
identity — silently defeating per-job token isolation. Fixed by prepending an
empty `-c credential.helper=` that RESETS the accumulated helper list, making
our ephemeral helper the only one git consults. Verified both directions with
real `git credential fill`.

Added two hermetic regression tests (`CREDENTIAL_HELPER_ARGS` describe block,
isolated via `GIT_CONFIG_GLOBAL`/`GIT_CONFIG_NOSYSTEM`):
- ours overrides a planted ambient helper → resolves to `x-access-token` + token;
- without the reset, the ambient helper wins → proves the reset is load-bearing.

Build clean; `npm test` → 6 files, **55 tests** (was 53), all pass.

`buildCloneUrl` (github.ts) left in place — still exported and unit-tested there;
workspace.ts simply no longer uses it.
