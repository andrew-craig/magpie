The core security deliverable of the epic (DISTRIBUTION.md §2, Design D). Blocked by the M7-0 feasibility spike.

Reviewer container: change `--network bridge/magpie-net` to `--network none` (loopback only, no interfaces) and add a per-job unix-socket mount `-v /run/magpie/jobs/<id>.sock:/run/gw.sock`. All other hardening flags unchanged (--cap-drop=ALL, --read-only --tmpfs /tmp, --user <uid>, --memory/--cpus/--pids-limit, .git-stripped ro /work, per-job virtual key).

Reviewer->gateway channel: a tiny in-container TCP->unix forwarder listens on 127.0.0.1:4000 and relays to /run/gw.sock; entrypoint.sh keeps writing models.json baseUrl=http://127.0.0.1:4000/v1. Forwarder holds no secret.

Gateway: proxy (data) plane now served over a unix socket instead of the magpie-net bridge IP; mgmt plane stays loopback (127.0.0.1) for mint/revoke. Gateway remains a HOST process (own user, 0600 key file) — key custody preserved, no docker inspect exposure. Virtual-key budget cap retained (still the hard cost cap).

Socket ownership: per-job socket connectable by the reviewer's --user uid and nothing else (0660, correct group, dir reachable only by the right users).

Fail-closed startup assertion in the reviewer entrypoint: assert no external route is reachable and the gateway socket is present; refuse to run otherwise (catches a mis-launch with the wrong --network).

DELETE scripts/setup-network.sh, the magpie-firewall.service unit, and the pinned 172.31.99.0/24 network contract — the reviewer has no network, so there is no bridge/iptables to provision.

Test: prove the reviewer has ZERO egress and can reach ONLY the gateway over the socket (no GitHub, no OpenRouter directly, no mgmt plane, no host, no internet, no DNS). With --network none this is provable/config-independent (not dependent on daemon.json).
