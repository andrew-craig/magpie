Turn Magpie from a single-server (Raspberry Pi) deployment into something other organisations can stand up. Primary target: a containerised docker compose stack, CONDITIONAL on preserving the per-run reviewer isolation (see DISTRIBUTION.md §2 for the isolation-equivalence analysis: egress lockdown moves from host iptables to a Docker internal-network topology with a two-plane gateway; reviewer stays an ephemeral hardened docker run, never a compose service; orchestrator uses DooD). Onboarding: docs-only this round (no App Manifest flow). Provider: OpenRouter-only (multi-provider stays M6-D). Host-native systemd kept as an advanced max-hardening path. Full strategy in DISTRIBUTION.md.
